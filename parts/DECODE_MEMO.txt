Mémo de décodage du PNG généré par `dump-all-and-rgba.ps1`

Contexte
- Le script prend le dump texte (`OutText`), l'écrit en bytes, puis le pad à un multiple de 4 (RGBA).
- Ensuite il écrit `OutRaw` (bytes) et peut ajouter des zéros supplémentaires pour atteindre `width*height*4` avant d'appeler ImageMagick :
  `magick -size ${width}x${height} -depth 8 rgba:in.raw out.png`

Pour décoder (reconstituer le texte depuis le PNG)
1) Extraire le flux RGBA brut depuis le PNG (ImageMagick) :
   - PowerShell / cmd (ImageMagick installé) :
     `magick convert in.png rgba:out.raw`
   - ou avec `magick` v7 :
     `magick in.png -depth 8 rgba:out.raw`

2) Retirer le padding ajouté :
   - Le script original a :
     - pad pour aligner la longueur à multiple de 4 (<= 3 octets), puis
     - padding éventuel pour faire `width*height*4` (remplissage par zéros)
   - Stratégie simple : supprimer les octets nuls de fin (trimming des `\x00`) jusqu'à retrouver un texte lisible.

Exemple Python minimal (Pillow) pour extraire et nettoyer :
```
from PIL import Image
b = Image.open('in.png').convert('RGBA').tobytes()
# retire les 0 de fin
b = b.rstrip(b'\x00')
with open('out.txt', 'wb') as f:
    f.write(b)
```

Remarques
- Si le texte peut contenir des zéros en fin de fichier valides, il faut conserver la longueur originale (non stockée dans le PNG). Dans ce dépôt, le script ajoute une ligne "Completed:" à la fin du dump — utiliser ce marqueur pour repérer la fin réelle du dump si nécessaire.
- Si ImageMagick manque, Pillow peut ouvrir le PNG et fournir les bytes RGBA (comme ci‑dessus).
